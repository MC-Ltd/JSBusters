      <h1><span class="secnum">6.1.7</span> The Object Type</h1>
      <p>Кожен екземпляр <dfn variants="is an Object,is not an Object" tabindex="-1">Object type ❮типу Об'єкт❯</dfn>, який також називають просто “an Object ❮Об'єкт❯”, представляє колекцію властивостей. Кожна властивість є або data property ❮властивістю даних❯, або accessor property <u>❮ >>> властивістю-аксесором / властивістю доступу <<< ❯</u>:</p>
      <ul>
        <li>
          <dfn variants="data properties" tabindex="-1">Data property ❮Властивість даних❯</dfn> пов'язує значення ключа зі <emu-xref href="#sec-ecmascript-language-types" id="_ref_1533"><a href="ecmascript-data-types-and-values.html#sec-ecmascript-language-types">ECMAScript language value ❮значенням мови ECMAScript❯</a></emu-xref> та встановлює Boolean ❮Булеві/Логічні❯ атрибути.
        </li>
        <li>
          <dfn variants="accessor properties" tabindex="-1">Accessor property ❮ >>> Властивість-аксесор / Властивість доступу <<< ❯</dfn> пов'язує значення ключа з однією або двома <u>>>> функціями-аксесорами / функціями доступу <<<</u> та встановлює Boolean  ❮Булеві/Логічні❯ атрибути. <u>>>> Функції-аксесори / Функції доступу  <<<</u> використовуються для зберігання або отримання <emu-xref href="#sec-ecmascript-language-types" id="_ref_1534"><a href="ecmascript-data-types-and-values.html#sec-ecmascript-language-types">ECMAScript language value ❮значення мови ECMAScript❯</a></emu-xref>, пов'язаного з властивістю.
        </li>
      </ul>
      <p>Властивості об'єкта однозначно ідентифікуються за допомогою <emu-xref href="#property-key" id="_ref_1535"><a href="ecmascript-data-types-and-values.html#property-key">property keys ❮ключів властивостей❯</a></emu-xref>. <dfn id="property-key" variants="property keys" oldids="sec-ispropertykey" tabindex="-1"><span id="sec-ispropertykey"></span>Property key ❮Ключ властивості❯</dfn> це або String ❮Рядок❯ або Symbol ❮Симбол❯. Всі Strings ❮Рядки❯ та Symbols ❮Симболи❯, включаючи порожній String ❮Рядок❯, є дійсними як <emu-xref href="#property-key" id="_ref_1536"><a href="ecmascript-data-types-and-values.html#property-key">property keys ❮ключи властивостей❯</a></emu-xref>. <dfn id="property-name" variants="property names" tabindex="-1">Property name ❮Ім'я властивості❯</dfn> – це є <emu-xref href="#property-key" id="_ref_1537"><a href="ecmascript-data-types-and-values.html#property-key">property key ❮ключ властивості❯</a></emu-xref> який <emu-xref href="#sec-ecmascript-language-types-string-type" id="_ref_1538"><a href="ecmascript-data-types-and-values.html#sec-ecmascript-language-types-string-type">is a String ❮є Рядком❯</a></emu-xref>.</p>
      <p><dfn id="integer-index" variants="integer indices,integer-indexed" tabindex="-1">Integer index ❮Цілочисельний індекс❯</dfn> – це <emu-xref href="#property-name" id="_ref_1539"><a href="ecmascript-data-types-and-values.html#property-name">property name ❮ім'я властивості❯</a></emu-xref> <var>n</var>, таке що <emu-xref aoid="CanonicalNumericIndexString" id="_ref_1540"><a href="abstract-operations.html#sec-canonicalnumericindexstring">CanonicalNumericIndexString</a></emu-xref>(<var>n</var>) повертає <emu-xref href="#integral-number" id="_ref_1541"><a href="notational-conventions.html#integral-number">integral Number ❮ціле Число❯</a></emu-xref> e <emu-xref href="#inclusive-interval" id="_ref_1542"><a href="notational-conventions.html#inclusive-interval">inclusive interval ❮включному інтервалі❯</a></emu-xref> від <emu-val>+0</emu-val><sub>𝔽</sub> до <emu-xref aoid="𝔽" id="_ref_1543"><a href="notational-conventions.html#%F0%9D%94%BD">𝔽</a></emu-xref>(2<span aria-hidden="true">**</span><sup>53</sup> - 1). <dfn id="array-index" variants="array indices" tabindex="-1">Array index ❮Індекс масиву❯</dfn> — це <emu-xref href="#integer-index" id="_ref_1544"><a href="ecmascript-data-types-and-values.html#integer-index">integer index ❮цілочисельний індекс❯</a></emu-xref> <var>n</var>, такий що <emu-xref aoid="CanonicalNumericIndexString" id="_ref_1545"><a href="abstract-operations.html#sec-canonicalnumericindexstring">CanonicalNumericIndexString</a></emu-xref>(<var>n</var>) повертає <emu-xref href="#integral-number" id="_ref_1546"><a href="notational-conventions.html#integral-number">integral Number ❮ціле Число❯</a></emu-xref> e <emu-xref href="#inclusive-interval" id="_ref_1547"><a href="notational-conventions.html#inclusive-interval">inclusive interval ❮включному інтервалі❯</a></emu-xref> від <emu-val>+0</emu-val><sub>𝔽</sub> до <emu-xref aoid="𝔽" id="_ref_1548"><a href="notational-conventions.html#%F0%9D%94%BD">𝔽</a></emu-xref>(2<span aria-hidden="true">**</span><sup>32</sup> - 2).</p>
      <emu-note><span class="note">Примітка</span><div class="note-contents">
        <p>Кожне невід’ємне <emu-xref href="#safe-integer" id="_ref_1549"><a href="numbers-and-dates.html#safe-integer">safe integer ❮безпечне ціле число❯</a></emu-xref> має відповідний <emu-xref href="#integer-index" id="_ref_1550"><a href="ecmascript-data-types-and-values.html#integer-index">integer index ❮цілочисельний індекс❯</a></emu-xref>. Кожне 32-бітне беззнакове <emu-xref href="#integer" id="_ref_1551"><a href="notational-conventions.html#integer">integer ❮ціле❯</a></emu-xref> крім <emu-eqn class="inline">2<span aria-hidden="true">**</span><sup>32</sup> - 1</emu-eqn> має відповідний <emu-xref href="#array-index" id="_ref_1552"><a href="ecmascript-data-types-and-values.html#array-index">array index ❮індекс масиву❯</a></emu-xref>. <emu-val>"-0"</emu-val> не є ні <emu-xref href="#integer-index" id="_ref_1553"><a href="ecmascript-data-types-and-values.html#integer-index">integer index ❮цілочисельний індекс❯</a></emu-xref>, ні <emu-xref href="#array-index" id="_ref_1554"><a href="ecmascript-data-types-and-values.html#array-index">array index ❮індекс масиву❯</a></emu-xref>.</p>
      </div></emu-note>
      <p><emu-xref href="#property-key" id="_ref_1555"><a href="ecmascript-data-types-and-values.html#property-key">Property keys ❮Ключі властивостей❯</a></emu-xref> використовуються для доступу до властивостей та їх значень. Існує два види доступу до властивостей: <em>get</em> ❮отримати❯ та <em>set</em> ❮встановити❯, що відповідають отриманню та присвоєнню значень відповідно. Властивості, доступні через доступ отримання та встановлення, включають як <em>власні властивості</em>, які є безпосередньою частиною об’єкта, так і <em>успадковані властивості</em>, які надаються іншим пов’язаним об’єктом через відношення успадкування властивостей. Успадковані властивості можуть бути як власними, так і успадкованими властивостями пов’язаного об’єкта. Кожна власна властивість об'єкта повинна мати ключове значення, яке відрізняється від ключових значень інших власних властивостей цього об'єкта.</p>
      <p>Усі об'єкти логічно є колекціями властивостей, але існує кілька форм об'єктів, які відрізняються своєю семантикою доступу та маніпулювання своїми властивостями. Будь ласка, дивіться <emu-xref href="#sec-object-internal-methods-and-internal-slots" id="_ref_100"><a href="ecmascript-data-types-and-values.html#sec-object-internal-methods-and-internal-slots">6.1.7.2</a></emu-xref> для визначення кількох форм об'єктів.</p>
      <p>Крім того, деякі об'єкти можна викликати; вони називаються функціями або <emu-xref href="#function-object" id="_ref_1556"><a href="ecmascript-data-types-and-values.html#function-object">function objects ❮функціональними об'єктами❯</a></emu-xref> а описані нижче. Усі функції в ECMAScript є членами Object type ❮типу Об'єкт❯.</p>
