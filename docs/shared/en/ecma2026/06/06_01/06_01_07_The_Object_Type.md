      <h1><span class="secnum">6.1.7</span> The Object Type</h1>
      <p>Each instance of the <dfn variants="is an Object,is not an Object" tabindex="-1">Object type</dfn>, also referred to simply as “an Object”, represents a collection of properties. Each property is either a data property, or an accessor property:</p>
      <ul>
        <li>
          A <dfn variants="data properties" tabindex="-1">data property</dfn> associates a key value with an <emu-xref href="#sec-ecmascript-language-types" id="_ref_1533"><a href="ecmascript-data-types-and-values.html#sec-ecmascript-language-types">ECMAScript language value</a></emu-xref> and a set of Boolean attributes.
        </li>
        <li>
          An <dfn variants="accessor properties" tabindex="-1">accessor property</dfn> associates a key value with one or two accessor functions, and a set of Boolean attributes. The accessor functions are used to store or retrieve an <emu-xref href="#sec-ecmascript-language-types" id="_ref_1534"><a href="ecmascript-data-types-and-values.html#sec-ecmascript-language-types">ECMAScript language value</a></emu-xref> that is associated with the property.
        </li>
      </ul>
      <p>The properties of an object are uniquely identified using <emu-xref href="#property-key" id="_ref_1535"><a href="ecmascript-data-types-and-values.html#property-key">property keys</a></emu-xref>. A <dfn id="property-key" variants="property keys" oldids="sec-ispropertykey" tabindex="-1"><span id="sec-ispropertykey"></span>property key</dfn> is either a String or a Symbol. All Strings and Symbols, including the empty String, are valid as <emu-xref href="#property-key" id="_ref_1536"><a href="ecmascript-data-types-and-values.html#property-key">property keys</a></emu-xref>. A <dfn id="property-name" variants="property names" tabindex="-1">property name</dfn> is a <emu-xref href="#property-key" id="_ref_1537"><a href="ecmascript-data-types-and-values.html#property-key">property key</a></emu-xref> that <emu-xref href="#sec-ecmascript-language-types-string-type" id="_ref_1538"><a href="ecmascript-data-types-and-values.html#sec-ecmascript-language-types-string-type">is a String</a></emu-xref>.</p>
      <p>An <dfn id="integer-index" variants="integer indices,integer-indexed" tabindex="-1">integer index</dfn> is a <emu-xref href="#property-name" id="_ref_1539"><a href="ecmascript-data-types-and-values.html#property-name">property name</a></emu-xref> <var>n</var> such that <emu-xref aoid="CanonicalNumericIndexString" id="_ref_1540"><a href="abstract-operations.html#sec-canonicalnumericindexstring">CanonicalNumericIndexString</a></emu-xref>(<var>n</var>) returns an <emu-xref href="#integral-number" id="_ref_1541"><a href="notational-conventions.html#integral-number">integral Number</a></emu-xref> in the <emu-xref href="#inclusive-interval" id="_ref_1542"><a href="notational-conventions.html#inclusive-interval">inclusive interval</a></emu-xref> from <emu-val>+0</emu-val><sub>𝔽</sub> to <emu-xref aoid="𝔽" id="_ref_1543"><a href="notational-conventions.html#%F0%9D%94%BD">𝔽</a></emu-xref>(2<span aria-hidden="true">**</span><sup>53</sup> - 1). An <dfn id="array-index" variants="array indices" tabindex="-1">array index</dfn> is an <emu-xref href="#integer-index" id="_ref_1544"><a href="ecmascript-data-types-and-values.html#integer-index">integer index</a></emu-xref> <var>n</var> such that <emu-xref aoid="CanonicalNumericIndexString" id="_ref_1545"><a href="abstract-operations.html#sec-canonicalnumericindexstring">CanonicalNumericIndexString</a></emu-xref>(<var>n</var>) returns an <emu-xref href="#integral-number" id="_ref_1546"><a href="notational-conventions.html#integral-number">integral Number</a></emu-xref> in the <emu-xref href="#inclusive-interval" id="_ref_1547"><a href="notational-conventions.html#inclusive-interval">inclusive interval</a></emu-xref> from <emu-val>+0</emu-val><sub>𝔽</sub> to <emu-xref aoid="𝔽" id="_ref_1548"><a href="notational-conventions.html#%F0%9D%94%BD">𝔽</a></emu-xref>(2<span aria-hidden="true">**</span><sup>32</sup> - 2).</p>
      <emu-note><span class="note">Note</span><div class="note-contents">
        <p>Every non-negative <emu-xref href="#safe-integer" id="_ref_1549"><a href="numbers-and-dates.html#safe-integer">safe integer</a></emu-xref> has a corresponding <emu-xref href="#integer-index" id="_ref_1550"><a href="ecmascript-data-types-and-values.html#integer-index">integer index</a></emu-xref>. Every 32-bit unsigned <emu-xref href="#integer" id="_ref_1551"><a href="notational-conventions.html#integer">integer</a></emu-xref> except <emu-eqn class="inline">2<span aria-hidden="true">**</span><sup>32</sup> - 1</emu-eqn> has a corresponding <emu-xref href="#array-index" id="_ref_1552"><a href="ecmascript-data-types-and-values.html#array-index">array index</a></emu-xref>. <emu-val>"-0"</emu-val> is neither an <emu-xref href="#integer-index" id="_ref_1553"><a href="ecmascript-data-types-and-values.html#integer-index">integer index</a></emu-xref> nor an <emu-xref href="#array-index" id="_ref_1554"><a href="ecmascript-data-types-and-values.html#array-index">array index</a></emu-xref>.</p>
      </div></emu-note>
      <p><emu-xref href="#property-key" id="_ref_1555"><a href="ecmascript-data-types-and-values.html#property-key">Property keys</a></emu-xref> are used to access properties and their values. There are two kinds of access for properties: <em>get</em> and <em>set</em>, corresponding to value retrieval and assignment, respectively. The properties accessible via get and set access includes both <em>own properties</em> that are a direct part of an object and <em>inherited properties</em> which are provided by another associated object via a property inheritance relationship. Inherited properties may be either own or inherited properties of the associated object. Each own property of an object must each have a key value that is distinct from the key values of the other own properties of that object.</p>
      <p>All objects are logically collections of properties, but there are multiple forms of objects that differ in their semantics for accessing and manipulating their properties. Please see <emu-xref href="#sec-object-internal-methods-and-internal-slots" id="_ref_100"><a href="ecmascript-data-types-and-values.html#sec-object-internal-methods-and-internal-slots">6.1.7.2</a></emu-xref> for definitions of the multiple forms of objects.</p>
      <p>In addition, some objects are callable; these are referred to as functions or <emu-xref href="#function-object" id="_ref_1556"><a href="ecmascript-data-types-and-values.html#function-object">function objects</a></emu-xref> and are described further below. All functions in ECMAScript are members of the Object type.</p>
